# 字符串和编码

## 字符编码

字符串在计算机中是一种数据类型，但字符串也有一个特殊的问题：**编码问题**。

### 计算机如何处理文本

- 计算机本身只能处理数字，而文本由字符组成。为了让计算机能够处理这些字符，必须先将它们转换成数字。
- 早期的计算机设计采用 **8 个比特（bit）= 1 字节（byte）**。
- 因此，计算机能够用 **1 个字节** 表示数字，1 个字节能表示的最大整数是 **255**（二进制 `11111111`，十进制 `255`）。
- 如果要表示更大的数字，就需要使用更多的字节。例如：
  - `2` 个字节表示的最大数字是 `65535`
  - `4` 个字节表示的最大数字是 `4294967295`

### 最初的字符编码：ASCII

- 最初的计算机只对 **127 个字符** 进行了编码，包括：
  - 大小写英文字母（`A-Z`、`a-z`）
  - 数字（`0-9`）
  - 一些常用符号（如空格、标点符号）
- 这套编码表被称为 **ASCII 编码**。
- ASCII 编码的特点是：
  - 每个字符用 **1 个字节** 表示
  - 例如：
    - 大写字母 `A` 的编码是 `65`
    - 小写字母 `z` 的编码是 `122`

### 扩展编码：支持更多字符

- 由于 ASCII 编码只能表示少量字符，因此对于中文、日文、韩文等非英语字符显然不够用。
- 中文最早常用 **GB2312** 编码，每个中文字符通常使用 **2 个字节** 表示，并尽量避免与 ASCII 编码冲突。
- 不同国家和地区也发展出了各自的编码标准，例如：
  - 日文：`Shift_JIS`
  - 韩文：`EUC-KR`

### 多语言冲突和乱码问题

- 由于不同国家和地区使用不同的字符编码标准，多语言混合时就可能出现**乱码**。
- 这种问题通常出现在：
  - 没有正确指定字符编码
  - 不同编码的文本被混合处理或显示
- 一旦程序或页面用错误的编码方式解析文本，就会出现无法正确显示字符的情况。

### Unicode 的出现

- 因此，**Unicode 字符集**应运而生。
- Unicode 把所有语言统一到一套编码体系里，这样就能够从根本上减少乱码问题。
- Unicode 标准也在不断发展，但常见的理解方式是：
  - 常用字符通常可以用 `2` 个字节表示
  - 如果遇到非常偏僻的字符，则可能需要 `4` 个字节
- 现代操作系统和大多数编程语言都直接支持 Unicode。

### ASCII 和 Unicode 的区别

- 可以先简单记住：
  - ASCII 编码使用 `1` 个字节
  - Unicode 编码通常使用 `2` 个字节
- 例如：
  - 字母 `A` 用 ASCII 编码是十进制 `65`，二进制 `01000001`
  - 字符 `0` 用 ASCII 编码是十进制 `48`，二进制 `00110000`
  - 注意：字符 `'0'` 和整数 `0` 是不同的概念
  - 汉字“中”已经超出了 ASCII 编码范围，用 Unicode 编码是十进制 `20013`，二进制 `01001110 00101101`
- 由此可以推测：
  - 如果把 ASCII 编码的 `A` 用 Unicode 编码表示，只需要在前面补 `0`
  - 因此，`A` 的 Unicode 编码是 `00000000 01000001`

### UTF-8 的出现

- 新的问题又出现了：如果统一使用 Unicode 编码，乱码问题虽然基本解决了，但如果文本内容几乎全是英文，那么 Unicode 会比 ASCII 占用更多存储空间，在存储和传输上不够划算。
- 因此，又出现了把 Unicode 转换为**可变长编码**的 **UTF-8** 编码。
- UTF-8 的特点是：
  - 一个 Unicode 字符会根据数值大小，被编码成 `1-6` 个字节
  - 常用英文字母通常编码成 `1` 个字节
  - 汉字通常编码成 `3` 个字节
  - 只有很生僻的字符才可能占用 `4-6` 个字节
- 如果传输的文本包含大量英文字符，使用 UTF-8 就可以节省空间。

### ASCII、Unicode 和 UTF-8 对比

| 字符 | ASCII      | Unicode             | UTF-8                        |
| ---- | ---------- | ------------------- | ---------------------------- |
| `A`  | `01000001` | `00000000 01000001` | `01000001`                   |
| `中` |            | `01001110 00101101` | `11100100 10111000 10101101` |

- 从上表还可以发现，UTF-8 还有一个额外的好处：
  - ASCII 编码实际上可以看作 UTF-8 编码的一部分
  - 因此，大量只支持 ASCII 编码的历史遗留软件，也可以在 UTF-8 编码下继续工作

### 现在计算机系统中的常见处理方式

- 搞清楚 ASCII、Unicode 和 UTF-8 的关系后，可以这样总结现代计算机系统常见的字符编码工作方式：
  - 在计算机内存中，统一使用 Unicode 编码
  - 当需要保存到硬盘或通过网络传输时，再转换为 UTF-8 编码
- 例如，用记事本编辑文件时：
  - 读取文件时，会把文件中的 UTF-8 字符转换为内存中的 Unicode 字符
  - 保存文件时，再把内存中的 Unicode 字符转换为 UTF-8 并写入磁盘
  - 浏览网页的时候，服务器会把动态生成的 Unicode 内容转换为 UTF-8 再传输到浏览器：
  - 所以你看到很多网页的源码上会有类似 `<meta charset="UTF-8" />` 的信息，表示该网页使用的是 UTF-8 编码。

## Python 的字符串

搞清楚了令人头疼的字符编码问题后，再来看 Python 的字符串。

- 在 Python 3 中，字符串以 Unicode 编码表示，因此天然支持多语言。

```python
print('包含中文的str')
```

输出：

```python
包含中文的str
```

### `ord()` 和 `chr()`

- 对于单个字符的编码，Python 提供了：
  - `ord()`：获取字符对应的整数表示
  - `chr()`：把整数编码转换为对应字符

```python
ord('A')
ord('中')
chr(66)
chr(25991)
```

输出：

```python
65
20013
'B'
'文'
```

- 如果知道字符的整数编码，也可以用十六进制形式表示字符串：

```python
'\u4e2d\u6587'
```

输出：

```python
'中文'
```

- 这两种写法是等价的。

### `str` 和 `bytes`

- Python 的字符串类型是 `str`，在内存中以 Unicode 表示。
- 如果要在网络上传输，或者保存到磁盘上，就需要把 `str` 转成以字节为单位的 `bytes`。
- `bytes` 类型用带 `b` 前缀的字符串表示：

```python
x = b'ABC'
```

- 要注意区分：
  - `'ABC'` 是 `str`
  - `b'ABC'` 是 `bytes`
- 它们看起来内容相同，但 `bytes` 的每个元素本质上是一个字节。

### `encode()`：把 `str` 变成 `bytes`

- 以 Unicode 表示的 `str` 可以通过 `encode()` 方法编码为指定的 `bytes`。

```python
'ABC'.encode('ascii')
'中文'.encode('utf-8')
'中文'.encode('ascii')
```

输出：

```python
b'ABC'
b'\xe4\xb8\xad\xe6\x96\x87'
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
UnicodeEncodeError: 'ascii' codec can't encode characters in position 0-1: ordinal not in range(128)
```

- 纯英文的 `str` 可以用 ASCII 编码为 `bytes`。
- 含有中文的 `str` 可以用 UTF-8 编码为 `bytes`。
- 含有中文的 `str` 无法用 ASCII 编码，因为中文超出了 ASCII 编码范围，因此 Python 会报错。
- 在 `bytes` 中，无法显示为 ASCII 字符的字节，会用 `\x##` 的形式显示。

### `decode()`：把 `bytes` 变成 `str`

- 如果从网络或磁盘读取到的是字节流，那么得到的数据就是 `bytes`。
- 要把 `bytes` 变成 `str`，需要使用 `decode()` 方法。

```python
b'ABC'.decode('ascii')
b'\xe4\xb8\xad\xe6\x96\x87'.decode('utf-8')
```

输出：

```python
'ABC'
'中文'
```

- 如果 `bytes` 中包含无法解码的字节，`decode()` 会报错：

```python
b'\xe4\xb8\xad\xff'.decode('utf-8')
```

输出：

```python
Traceback (most recent call last):
  ...
UnicodeDecodeError: 'utf-8' codec can't decode byte 0xff in position 3: invalid start byte
```

- 如果只有少量无效字节，可以传入 `errors='ignore'` 忽略错误：

```python
b'\xe4\xb8\xad\xff'.decode('utf-8', errors='ignore')
```

输出：

```python
'中'
```

### `len()` 统计字符数和字节数

- 计算 `str` 包含多少个字符，可以用 `len()`：

```python
len('ABC')
len('中文')
```

输出：

```python
3
2
```

- 如果换成 `bytes`，`len()` 计算的就是字节数：

```python
len(b'ABC')
len(b'\xe4\xb8\xad\xe6\x96\x87')
len('中文'.encode('utf-8'))
```

输出：

```python
3
6
6
```

- 可见：
  - `str` 的 `len()` 统计的是字符数
  - `bytes` 的 `len()` 统计的是字节数
  - `1` 个中文字符经过 UTF-8 编码后通常占 `3` 个字节
  - `1` 个英文字符通常只占 `1` 个字节

### 关于编码的建议

- 在操作字符串时，经常会遇到 `str` 和 `bytes` 的互相转换。
- 为了避免乱码，通常应当优先统一使用 **UTF-8** 对 `str` 和 `bytes` 进行转换。

### Python 源码的编码

- Python 源代码本身也是文本文件。
- 因此，当源代码中包含中文时，保存文件时应确保使用 UTF-8 编码。
- 为了让 Python 解释器按 UTF-8 读取源代码，通常会在文件开头写上：

```python
#!/usr/bin/env python3
# -*- coding: utf-8 -*-
```

- 第一行注释是告诉 Linux 或 macOS 系统：这是一个 Python 可执行程序。
- Windows 一般会忽略这行注释。
- 第二行注释是告诉 Python 解释器：按 UTF-8 编码读取源代码，否则源代码中的中文可能会出现乱码。
- 需要注意的是：
  - 声明了 UTF-8 编码，并不意味着 `.py` 文件本身就一定是 UTF-8
  - 还必须确保你的文本编辑器确实使用 UTF-8 保存该文件

## 格式化

最后一个常见问题是：如何输出格式化字符串。

- 例如，经常会输出类似“亲爱的 xxx 你好！你 xx 月的话费是 xx，余额是 xx”这样的内容。
- 由于其中的数据来自变量，因此需要一种简便的字符串格式化方式。

### `%` 格式化

- 在 Python 中，较传统的格式化方式与 C 语言类似，使用 `%` 实现。

```python
'Hello, %s' % 'world'
'Hi, %s, you have $%d.' % ('Michael', 1000000)
```

输出：

```python
'Hello, world'
'Hi, Michael, you have $1000000.'
```

- `%s` 表示用字符串替换
- `%d` 表示用整数替换
- 有几个占位符，后面就要跟几个变量或值，并且顺序要对应
- 如果只有一个占位符，括号可以省略

常见占位符如下：

| 占位符 | 替换内容     |
| ------ | ------------ |
| `%d`   | 整数         |
| `%f`   | 浮点数       |
| `%s`   | 字符串       |
| `%x`   | 十六进制整数 |

- 格式化整数和浮点数时，还可以指定补 `0` 和保留小数位数：

```python
print('%2d-%02d' % (3, 1))
print('%.2f' % 3.1415926)
```

- 如果不确定该用什么占位符，`%s` 基本总是可以工作，因为它会把任意数据类型转换成字符串：

```python
'Age: %s. Gender: %s' % (25, True)
```

输出：

```python
'Age: 25. Gender: True'
```

- 如果字符串里的 `%` 本身只是普通字符，就需要写成 `%%`：

```python
'growth rate: %d %%' % 7
```

输出：

```python
'growth rate: 7 %'
```

### `format()`

- 另一种格式化字符串的方法是使用字符串的 `format()` 方法。
- 它会用传入参数依次替换字符串中的占位符 `{0}`、`{1}` 等。

```python
'Hello, {0}, 成绩提升了 {1:.1f}%'.format('小明', 17.125)
```

输出：

```python
'Hello, 小明, 成绩提升了 17.1%'
```

### `f-string`

- 现在更常用的方式是以 `f` 开头的字符串，也就是 **f-string**。
- 如果字符串中包含 `{xxx}`，Python 会直接用对应变量的值替换它。

```python
r = 2.5
s = 3.14 * r ** 2
print(f'The area of a circle with radius {r} is {s:.2f}')
```

输出：

```python
The area of a circle with radius 2.5 is 19.62
```

- 上述代码中：
  - `{r}` 会被变量 `r` 的值替换
  - `{s:.2f}` 会被变量 `s` 的值替换
  - `.2f` 表示保留两位小数
